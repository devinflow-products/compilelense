CompileLens — Getting Started

A live IntelliJ IDEA dashboard for Java classes that do not compile—see every problem, jump to the source, and watch the list update as you fix code.

What is CompileLens? CompileLens aggregates editor diagnostics, the Problems view, and compiler output into one tool window so you can track unresolved and uncompiled Java classes across your project without hunting through individual editor tabs.

Requirements

RequirementDetails
IDEIntelliJ IDEA 2025.3+ (Ultimate or compatible build with Java support)
LanguageJava source files (.java) in configured source roots
PluginsJava support and Kotlin plugin (declared dependencies)
ProjectsMaven, Gradle, or plain Java modules

CompileLens analyzes Java sources only; Kotlin-only files are out of scope.

Installation

From source (developers)

  1. Open this repository in IntelliJ IDEA and wait for Gradle sync.
  2. Run the Run Plugin configuration, or execute:
# macOS / Linux
./gradlew runIde

# Windows
gradlew.bat runIde

A sandbox IDE starts with CompileLens installed. Open a Java project to try it.

From JetBrains Marketplace

When published, install CompileLens via Settings → Plugins → Marketplace, then restart the IDE.

Open the dashboard

  • Tool window — Click CompileLens on the right tool window stripe.
  • View menu — View → CompileLens
  • Tools menu — Tools → CompileLens

The window shows a sidebar (summary and filters) and a main table (classes with issues).

First run

  1. CompileLens waits until your project leaves dumb mode (indexing complete).
  2. An initial re-scan runs: rebuilds open projects and populates the table.
  3. The sidebar shows totals, per-module counts, and last scan time.
After the first scan, the list updates automatically as you fix errors—rows disappear when the IDE clears diagnostics for that file.

Navigation

  • Double-click a row → open at the first error line.
  • Click a line number → open at that line.
  • Click Folder → highlight the parent folder in the project tree.

Use the search box to filter by class, package, folder, or module. Sort by folder or class name.

Live updates

CompileLens refreshes the dashboard without manual action:

  • While editing — Fixed files are removed after code analysis finishes (typically within about a second).
  • Save / VFS — Saving files or editing pom.xml / Gradle build files triggers a scan.
  • Problems view — Appearing or disappearing problems update the list.
  • After build — Compiler messages from the last rebuild are included; stale per-file build errors clear when the editor shows no issues.
Use Re-scan when you need a full Build → Rebuild Project pass—for example after large dependency or module structure changes.

Issue types

TypeTypical cause
Unresolved importMissing or incorrect import
Class not foundType not on the classpath
Missing dependencyPackage or symbol requires a library or module dependency
Compilation errorSyntax, type errors, and other javac/IDE diagnostics

Recommended workflow

  1. Open your Java project and wait for indexing.
  2. Open CompileLens from the right tool window stripe.
  3. Use the sidebar module breakdown to prioritize busy modules.
  4. Click a row or line number to jump to the code and fix the error.
  5. Confirm the row disappears when the red underline clears.
  6. Click Re-scan after POM/Gradle or module changes if the list seems stale.

Troubleshooting

SymptomWhat to try
Dashboard empty but editor shows errors Wait for indexing; click Re-scan; confirm the file is under a Java source root
Fixed file still listed Wait for analysis to finish; click Re-scan if needed
No updates while editing Ensure .java file and project is not indexing (dumb mode)
Re-scan is slow Expected on large multi-module projects (full rebuild per open project)
Plugin not visible Settings → Plugins — enable CompileLens and restart

Logs: Search idea.log for CompileLens in development sandboxes.

Privacy and scope

  • Analysis uses local project data only (PSI, highlights, compiler output in the IDE).
  • Scans cover open projects and Java sources in source roots.
  • The plugin does not send data to external services.
